# Bunker 7 — Онлайн игра на выживание

Production-ready веб-приложение для онлайн-игры "Бункер" с полной функциональностью мультиплеера.

## Особенности

- ✅ **Реальный онлайн мультиплеер** с WebSocket (Socket.IO)
- ✅ **База данных SQLite** для хранения комнат и игроков
- ✅ **Полная панель ведущего** с контролем всех аспектов игры
- ✅ **8 случайных характеристик** для каждого игрока
- ✅ **Система голосования** с автоматическим подсчётом
- ✅ **Таймеры раундов** (15, 30, 60, 120 секунд)
- ✅ **Чат в реальном времени**
- ✅ **Переподключение** при обновлении страницы
- ✅ **6 сценариев катастроф** с случайным выбором
- ✅ **Адаптивный дизайн** для ПК, планшетов и мобильных
- ✅ **Минималистичный UI** в черно-зеленом стиле

## Технологии

- **Frontend:** React 19, TypeScript, Tailwind CSS, Socket.IO Client
- **Backend:** Node.js 22, Express, Socket.IO, Better-SQLite3
- **Routing:** React Router v7
- **Build:** Vite 7

## Установка и запуск

### Разработка

```bash
# Установить зависимости
npm install

# Запустить сервер и клиент одновременно
npm run dev
```

Сервер будет доступен на `http://localhost:8080`, клиент на `http://localhost:5173`

### Production сборка

```bash
# Собрать фронтенд
npm run build

# Запустить production сервер
npm start
```

Сервер будет слушать порт из переменной окружения `PORT` или `8080` по умолчанию.

## Как играть

### Для ведущего:

1. Создайте комнату, введите своё имя
2. Поделитесь кодом комнаты с игроками
3. Дождитесь подключения всех игроков (минимум 2)
4. Нажмите "Начать игру"
5. Управляйте игрой через панель ведущего:
   - Переключайте фазы игры
   - Запускайте таймеры (15/30/60/120 сек)
   - Назначайте текущего спикера
   - Запускайте/завершайте голосования
   - Раскрывайте/изменяйте карты игроков
   - Исключайте/восстанавливайте игроков

### Для игроков:

1. Получите код комнаты от ведущего
2. Введите код и своё имя
3. Ожидайте начала игры в лобби
4. В Discord (голосовой канал) ведущий даёт команды
5. В браузере раскрывайте свои карточки по команде ведущего
6. Аргументируйте свою ценность для бункера
7. Голосуйте за исключение других игроков

### Правила раскрытия карт:

- **Обязательно каждый раунд:** 1 профессия + 2 любые другие характеристики
- Ведущий контролирует это в Discord
- Игроки сами кликают на карты в браузере для раскрытия
- Ведущий может раскрыть/изменить любую карту через панель

## API Endpoints

- `GET /health` - Health check (возвращает статус сервера)
- `GET /*` - SPA fallback (раздаёт index.html для React Router)

## Socket.IO Events

### Client → Server:
- `create_room` - Создать комнату
- `join_room` - Войти в комнату
- `reconnect_room` - Переподключиться
- `start_game` - Начать игру (host)
- `set_phase` - Установить фазу (host)
- `start_timer` - Запустить таймер (host)
- `start_voting` - Начать голосование (host)
- `end_voting` - Завершить голосование (host)
- `vote` - Проголосовать (player)
- `player_reveal_card` - Раскрыть свою карту (player)
- `reveal_card` - Раскрыть чужую карту (host)
- `change_card_value` - Изменить значение карты (host)
- `eliminate_player` - Исключить игрока (host)
- `chat_message` - Отправить сообщение в чат

### Server → Client:
- `room_updated` - Обновление состояния комнаты (public)
- `room_host_updated` - Обновление для ведущего (с полными данными)
- `chat_message` - Новое сообщение в чате
- `kicked` - Игрок выгнан из комнаты

## Дополнительные функции ведущего

- **Кубик удачи:** Перегенерация всех карт игрока
- **Изменение характеристики:** Ручное изменение значения любой карты
- **Смена катастрофы:** Выбор другого сценария
- **Отмена действия:** Восстановление исключённого игрока
- **Кик игрока:** Удаление из комнаты
- **Перезапуск игры:** Сброс и возврат в лобби

## Сценарии катастроф

1. **Ядерная война** - 25 лет изоляции
2. **Смертельная пандемия** - 15-20 лет
3. **Падение астероида** - 30 лет
4. **Восстание ИИ** - неопределённый срок
5. **Климатическая катастрофа** - 50+ лет
6. **Зомби-апокалипсис** - неопределённый срок
